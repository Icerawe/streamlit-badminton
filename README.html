🏸 เชียงใหม่แบดมินตัน Ranking System

วิธีรัน

# ใช้ uv (แนะนำ)
uv run streamlit run app.py

# หรือใช้ pip
pip install streamlit pandas openpyxl plotly
streamlit run app.py

Match Categories (ประเภทการแข่งขัน)

แต่ละประเภทมีหลาย pair (doubles) โดย rank ของผู้เล่นต้องอยู่ใน range ที่กำหนด

ประเภทจับคู่แบบที่ 1จับคู่แบบที่ 2จับคู่แบบที่ 3
BG1BG – BG––
BG2/NBN- – N-BG – N–
NBG – SN- – N+N – N
N/SN- – P-N – SN+ – N+
SS – SP- – N+–
PP- – P+P – P–
⚠️ ถ้า rank ห่างกันเกิน 3 ระดับ ระบบจะเตือนว่า "ไม่มีประเภทที่เหมาะสม / มือห่างเกินไป"

หน้าต่างๆ

🏠 Home

  • สถิติรวม: จำนวนผู้เล่น, จำนวนทีม, P+ count
  • กราฟ rank distribution
  • ภาพรวมทีมพร้อมจำนวนสมาชิก
  • ระดับ Rank ทั้งหมด (rank cards)
  • ตารางประเภทการแข่งขัน (จับคู่แบบที่ 1, 2, 3)
  • บันไดระดับ rank

📊 ตารางอันดับ

  • แสดงผู้เล่นจัดกลุ่มตาม Rank กรองตามทีม
  • 🚩 flag เมื่อ net protest votes ≥ 5

🔍 Match Finder

  • Mode A: เลือก 2 คน → แสดง match categories ที่ใช้ได้
  • Mode B: เลือกทีม → แสดง pairings ที่เป็นไปได้
  • เตือนเมื่อ rank ห่างเกินไป

✊ ประท้วง / โหวต

  • โหวต +1/-1 ระดับ ได้คนละ 1 ครั้งต่อผู้เล่น (session-based)
  • ใส่เหตุผล (text) หรือ YouTube URL ประกอบได้

🔧 Admin

  • รหัสผ่าน: badmin1234
  • เพิ่ม/แก้ไข/ลบผู้เล่น พร้อม team field
  • อัปโหลด Excel (format: ทีม / ชื่อเล่น / ระดับ)
  • จัดการเสียง protest → ปรับ rank หรือล้าง votes
  • Export CSV รายชื่อทั้งหมด

อัปโหลดรายชื่อ (Excel Upload)

ดาวน์โหลด template จาก Admin → Upload Excel หรือใช้ไฟล์ data/upload_template.xlsx

Database Schema

players:  id, name (UNIQUE), rank, team, created_at, updated_at
protests: id, player_id (FK), voter_session, direction (+1/-1),
          reason, youtube_url, created_at

Deploy บน Streamlit Community Cloud

  1. Push โค้ดขึ้น GitHub
  2. ไปที่ https://share.streamlit.io → เลือก repo → app.py → Deploy
⚠️ หมายเหตุ: SQLite data จะ reset เมื่อ redeploy
ถ้าต้องการถาวรให้เปลี่ยนเป็น PostgreSQL หรือ Supabase
